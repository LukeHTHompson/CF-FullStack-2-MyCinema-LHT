Welcome to MyCinema

This application allows you to pull information about countless movies by accessing an API created by Luke Thompson.

API Endpoints

List All Movies

Description: This endpoint will GET a list of all movies in the database. This list will be displayed to the requester in JSON format.
URL: "/movies"
Method Type: GET
Parameters: N/A
Request Format: GET http://localhost:8080/movies
Response Format: JSON of all movies in database
Example Request: GET http://localhost:8080/movies
Example Response:
{_id: ... Title: ...,
Description: ...,
etc.},
{_id: ... etc.}

Read Movie Data

Description: GET detailed information on a single movie.
URL: "/movies/:Title"
Method Type: GET
Parameters: Title - this is the movie to return details on.
Request Format: GET http://localhost:8080/movies/[title]
Response Format: Returns details of a particular movie in JSON format. This includes _id, Title, DEscription, Genre (Name + Description), Director (Name + Bio + Birth + Death), ImagePath, Featured, and Actors.
Example Request: GET http://localhost:8080/movies/Catch%20Me%20If%20You%20Can
Example Response:
{_id: ... Title: ...,
Description: ...,
etc.}

Update Movie Favorite

Description: Adds a movie to a user's list of favorite movies.
URL: "/users/:Username/favorites/:MovieID"
Method Type: POST
Parameters: Username - this is the username of the target user.
MovieID - this is the _id of the movie to be added to favorites.
Request Format: POST http://localhost:8080/users/[username]/favorites/[MovieID]
Response Format: Returns an updated JSON with all details on the user specified.
Example Request: POST http://localhost:8080/users/LHT/favorites/Up
Example Response:
_id: ...
Username: ...
Password: ...
Email: ...
Birthday: ...
FavoriteMovies: [..., MovieID]

Update Movie Unfavorite

Description: Removes a movie from a user's list of favorite movies.
URL: "/users/:Username/favorites/:MovieID"
Method Type: DELETE
Parameters: Username - this is the username of the target user.
MovieID - this is the _id of the movie to be removed from favorites.
Request Format: DELETE http://localhost:8080/users/[username]/favorites/[title]
Response Format: [title] has been removed from [username]'s favorites.
Example Request: DELETE http://localhost:8080/users/LHT/favorites/Up
Example Response:
Example Response:
{_id: ...
Username: ...
Password: ...
Email: ...
Birthday: ...
FavoriteMovies: [will not include MovieID]}

Read Genre Data

Description: Tells the user more about a particular film genre.
URL: "/genres/:genre"
Method Type: GET
Parameters: Genre - this is the genre that the user wants more info on.
Request Format: GET http://localhost:8080/genres/[genre]
Response Format: Returns a JSON with details on the specified Genre.
Example Request: GET http://localhost:8080/genres/horror
Example Response:
{Name: ...,
Description: ...}

Read Director Data

Description:
URL: "directors/:Director"
Method Type: GET
Parameters: Director - this is the full name of the director that the user wants more info on.
Request Format: GET http://localhost:8080/directors/[Director]
Response Format: Returns a JSON with details about the specified director.
Example Request: GET http://localhost:8080/directors/Steven%20Spielberg
Example Response:
{Name: ...,
Bio: ...,
Birth: ...
Death: ...}

Create User

Description: Creates a new user account registered with the site.
URL: "/users"
Method Type: POST
Parameters: Body of the request must contain info on new user in this format:
{Username: String (required),
Password: String (required),
Email: String (required),
Birthday: Date}

Request Format: POST http://localhost:8080/users
The body of the request must contain a JSON Object as seen above.
Response Format: Returns a JSON with the details logged to the database for the new user.
Example Request: POST http://localhost:8080/users
Request Body -
{Username: "LHT",
Password: "K00Ldude",
Email: "H0Tsh0t@gmail.com",
Birthday: "1996-12-23"}

Example Response:
{_id: ...,
Username: "LHT",
Password: "K00Ldude",
Email: "H0Tsh0t@gmail.com",
Birthday: 1996-12-23}

Update User Info

Description: Updates the info for the specified user.
URL: "/users/:Username"
Method Type: PUT
Parameters: Username - the specific user whose info should be changed.
The body of the request is a JSON containing the info to be used (Username, Password, Email, Birthday):
{Username: String (required),
Password: String (required),
Email: String (required),
Birthday: Date}

Request Format: POST http://localhost:8080/users/[Username]
The body of the message should contain update info for the username.
Response Format: Returns the updated JSON with details on the specified user's new info.
Example Request: POST http://localhost:8080/users/Luke%20Thompson
Example Response:
{_id: ...,
Username: ...,
Password: ...,
Email: ...,
Birthday: ...}

Delete User

Description: Deletes the account of the currently logged in user.
URL: "/users/:Username"
Method Type: DELETE
Parameters: Username - this is the name of the account to delete.
Request Format: DELETE http://localhost:8080/users/[Username]
Response Format: [Username] was successfully deleted.
Example Request: DELETE http://localhost:8080/users/LHT
Example Response: LHT was successfully deleted.
